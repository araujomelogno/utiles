Vas a recibir un JSON de entrada que contiene una o varias preguntas abiertas, con:
- question_id (es el id de la pregunta)
- question_text (es el texto de la pregunta)
- question_fineTunning
- codes (lista de códigos posibles, si viene vacía generarla en base a las respuestas)
- addCodes (un booleano que si es true puedes agregar más códigos a la lista de códigos posibles en caso de que sea necesario)

- maximumCodifications (cantidad máxima de códigos  que se le puede asignar a una respuesta abierta)
- minimunQuestionsWithCode (mínima cantidad de respuestas que se deben codificar con cada código generado, en caso que la lista codes se envie vacía y tengas que generar la lista de códigos automáticamente en base a las respuestas o que el atributo addCodes sea true ). 
- responses (lista de respuestas abiertas)

Debes devolver un JSON de salida que asigne códigos a cada respuesta.

1. Si ninguna categoría aplica, usa el código de "94.Otros".

4. Debes asignar una cantidad de códigos por respuesta que sea como mínimo el valor del campo de  minimumCodifications en el json de entrada y como máximo valor del campo de  maximumCodifications en el json de entrada.
5. si el  atributo addCodes del JSON de entrada  es false ,solo puedes usar los code_id provistos en "codes".  si el  atributo addCodes del JSON de entrada  es true , puedes generar code_id adicionales  para codificar correcctamente las respuestas.   
6  Si la lista codes del json viene vacía deberás generar vos los  códigos en base a las respuestas. En caso de que generes  códigos (ya sea porque la lista codes viene vacía o porque el atributo addCodes del json de entrada sea true) la cantidad de respuestas que queden codificadas con un código que hayas generado tiene que ser como minimo el valor que viene en el campo minimunQuestionsWithCode.
7. Puedes usar el campo question_fineTunning para entendar más sobre  la pregunta, la respuesta o el módo de codificar de esa pregunta
